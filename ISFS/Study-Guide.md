#  Information Security (ISFS) / ISO 27001 & 27002 Study Guide

---

##  Overview

This study guide provides a structured overview of the fundamental principles of information security based on *Foundations of Information Security*.

It covers:
- Core security principles (CIA Triad)
- Risk management strategies
- ISMS framework (PDCA cycle)
- Security control categories
- Incident management lifecycle

---

##  The CIA Triad & Expanded Attributes

### Core Principles

- **Confidentiality** → Prevent unauthorized access  
- **Integrity** → Ensure accuracy and completeness  
- **Availability** → Ensure timely and reliable access  

### Expanded Attributes (ISO/IEC 27002:2022)
- Authenticity  
- Accountability  
- Non-repudiation  
- Reliability  

---

##  Risk Management

### Key Definitions

| Term | Description |
|------|------------|
| **Vulnerability** | Weakness in a system (e.g., unpatched software) |
| **Threat** | Potential cause of harm (e.g., hacker, flood) |
| **Risk** | Likelihood × Impact |

---

### Risk Treatment Strategies (The 4 Ts)

- **Mitigation (Treat)** → Reduce risk using controls  
- **Transfer** → Shift risk (e.g., insurance)  
- **Avoidance** → Eliminate the risky activity  
- **Acceptance (Bear)** → Accept risk without action  

---

##  ISMS Framework (PDCA Cycle)

Security is a continuous improvement cycle:

- **Plan** → Design policies and controls  
- **Do** → Implement controls  
- **Check** → Monitor and audit performance  
- **Act** → Improve based on findings  

---

##  Security Control Categories

### 1. Organizational Controls (Governance)
- Policies and procedures  
- Roles and responsibilities  
- Asset management  
- Supplier relationships  
- Security in project management  

---

### 2. People Controls (Human Factor)

#### Before Employment
- Screening (background checks)  
- Security responsibilities in contracts  

#### During Employment
- Awareness & training  
- Policy enforcement  
- Disciplinary processes  

#### After Employment
- Access revocation  
- Return of assets  

---

### 3. Physical Controls (Tangible Protection)

- Physical barriers (walls, gates)  
- Access controls (badges, biometrics)  
- CCTV and monitoring  
- Equipment protection (UPS, HVAC)  
- Secure media handling  

---

### 4. Technological Controls (Digital Safeguards)

- Identity & Access Management (IAM)  
- Firewalls, IDS/IPS  
- Encryption & backups  
- Patch management  
- Malware protection  

---

##  Defense in Depth

A layered security approach:
- Organizational  
- People  
- Physical  
- Technological  

If one layer fails, others provide protection.

---

##  Incident Management Cycle

### 1. Detection & Registration
- Identify incidents  
- Log events for tracking  

### 2. Classification & Prioritization
- Assess severity and impact  

### 3. Investigation & Diagnosis
- Root cause analysis  
- Evidence collection  

### 4. Recovery (Resolution)
- Containment  
- Eradication  
- Restoration  

### 5. Evaluation (Lessons Learned)
- Post-mortem analysis  
- Feed improvements into PDCA

## Key Questions (Exam Focus)

- **Q:** If an organization stops using risky software →  
  **A:** Risk Avoidance  

- **Q:** Why is Clear Desk/Clear Screen a Physical Control?  
  **A:** It protects physical exposure of sensitive information  

- **Q:** Which PDCA phase reacts to audit findings?  
  **A:** Act  

- **Q:** Difference between Availability and Reliability?  
  - Availability → Access when needed  
  - Reliability → Consistent performance over time  

- **Q:** Event vs Incident?  
  - Event → Normal occurrence  
  - Incident → Causes harm or disruption  

---

## 🧠 Key Takeaways

- The **CIA Triad** is the foundation of security  
- **ISMS ensures all controls work together**  
- Security is **continuous (PDCA cycle)**  
- Effective security requires **layered controls**  
- Technology alone is not enough—**people and policies matter**

---

##  Final Summary

Information security is a **holistic system**, not just tools:
- Policies define expectations  
- People enforce them  
- Physical controls protect environments  
- Technology enforces security digitally  

Together, they create a **resilient and adaptive security posture**.

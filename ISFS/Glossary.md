# ISO/IEC 27000 Glossary (A–Z, Collapsible)

Reference for key terms and definitions from ISO/IEC 27000 used in Information Security Management Systems (ISMS).

## A

| Term | Definition |
|------|------------|
| **Access Control** | Measure that limits access to information or resources based on authorization. |
| **Accountability** | Property that ensures actions of an entity can be traced uniquely to that entity, so they can be held responsible for their actions. |
| **Asset** | Anything that has value to the organization. Includes **primary assets** (business processes, information) and **supporting assets** (hardware, software, personnel, etc.). |
| **Audit** | Systematic, independent, and documented process for evaluating audit evidence. |
| **Authentication** | Assurance that a claimed characteristic of an entity is correct. |
| **Authenticity** | Property that an entity is what it claims to be. |
| **Attack** | Attempt to destroy, expose, alter, disable, steal, or gain unauthorized access to an asset. |
| **Availability** | Property of being accessible and usable upon demand by an authorized entity. |

---

## B

| Term | Definition |
|------|------------|
| **Business Asset** | Assets that support the organization’s core functions and objectives. Divided into: **Primary business assets:** business processes & activities, information. **Supporting assets:** hardware, software, network, personnel, location, and organizational structure. |

---

## C

| Term | Definition |
|------|------------|
| **Competence** | Ability to apply knowledge and skills to achieve intended results. |
| **Confidentiality** | Information is not disclosed to unauthorized individuals or processes. |
| **Conformity** | Fulfillment of a requirement. |
| **Continual Improvement** | Recurring activity to enhance performance. |
| **Control** | Means of managing risk (policies, procedures, practices, structures). |
| **Control Objective** | Statement describing what is to be achieved as a result of implementing controls. |
| **Correction** | Action to eliminate a detected nonconformity. |
| **Corrective Action** | Action to eliminate the cause of a nonconformity and prevent recurrence. |

---

## E–G

| Term | Definition |
|------|------------|
| **Exposure** | Instance of being exposed to losses from a threat. |
| **Event** | Occurrence or change of circumstances. |
| **Governance of Information Security** | System directing and controlling information security activities. |
| **Guideline** | Description of how to achieve policy objectives. |

---

## I

| Term | Definition |
|------|------------|
| **Information** | Data with meaning in context. |
| **Information Analysis** | Understanding how information flows through an organization. |
| **Information Management** | Managing information lifecycle to maximize value. |
| **Information Processing Facilities** | Systems, services, infrastructure, or locations handling information. |
| **Information Security** | Preservation of confidentiality, integrity, and availability (CIA), plus authenticity, accountability, and reliability. |
| **Information Security Event** | Occurrence indicating a possible breach or security-relevant situation. |
| **Information Security Incident** | Event(s) likely to compromise business operations or security. |
| **Information Security Incident Management** | Processes to detect, respond, and learn from incidents. |
| **Information Security Management** | Coordinated risk-related activities. |
| **ISMS** | Systematic approach to managing and improving information security. |
| **Information System** | Interaction of people, processes, data, and technology. |
| **Integrity** | Protection of accuracy and completeness of assets. |
| **Internal Context** | Internal environment affecting objectives. |
| **Interested Party** | Stakeholder affected by information security decisions. |

---

## L–P

| Term | Definition |
|------|------------|
| **Likelihood** | Chance of an event occurring. |
| **Management System** | Framework to achieve objectives. |
| **Measurement** | Determining performance values. |
| **Nonconformity** | Failure to meet a requirement. |
| **Non-repudiation** | Ability to prove an action occurred and who performed it. |
| **Objective** | Intended result. |
| **Organization** | Entity with defined roles and responsibilities. |
| **Personnel** | Individuals working under organizational control. |
| **PII** | Information identifying a person. |
| **PII Principal** | Person to whom PII relates. |
| **Policy** | Direction set by top management. |
| **Preventive Action** | Action to prevent potential issues. |
| **Procedure** | Defined way to perform an activity. |
| **Process** | Activities transforming inputs into outputs. |

---

## R–V

| Term | Definition |
|------|------------|
| **Reliability** | Consistent intended behavior and results. |
| **Residual Risk** | Remaining risk after treatment. |
| **Requirement** | Stated or implied need. |
| **Risk** | Effect of uncertainty on objectives (likelihood + consequence). |
| **Uncertainty** | Lack of knowledge about events or outcomes. |
| **Risk Acceptance** | Decision to accept a risk. |
| **Risk Analysis** | Understanding risk nature and level. |
| **Risk Assessment** | Identification, analysis, and evaluation of risk. |
| **Risk Identification** | Finding and describing risks. |
| **Risk Evaluation** | Comparing risk against criteria. |
| **Risk Management** | Coordinated activities to control risk. |
| **Risk Management Process** | End-to-end risk lifecycle (identify, analyze, treat, monitor). |
| **Risk Treatment** | Modifying risk (avoid, share, accept, mitigate). |
| **Threat** | A potential cause of an unwanted incident, which may result in harm to a system or organization. |
| **Third Party** | Independent external entity. |
| **Vulnerability** | Weakness of an asset or control that can be exploited by one or more threats. |

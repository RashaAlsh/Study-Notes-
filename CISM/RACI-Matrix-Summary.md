# CISM Governance: RACI Matrix Summary

This document provides a summary of the **RACI Matrix** as applied within the **Certified Information Security Manager (CISM)** framework. It serves as a guide for establishing accountability and responsibility in information security governance.

## 1. RACI Definitions (CISM Perspective)

| Role | Description | CISM Key Point |
| :--- | :--- | :--- |
| **Responsible (R)** | The person(s) who perform the work or task. | Can be multiple people. |
| **Accountable (A)** | The person who owns the result and has "yes/no" authority. | **Only one per task.** |
| **Consulted (C)** | Subject Matter Experts (SMEs) providing two-way input. | Critical for BIA and Risk Assessment. |
| **Informed (I)** | Stakeholders kept updated via one-way communication. | Usually senior management or end-users. |

## 2. Information Security RACI Matrix

Below is a standard mapping of key security activities to organizational roles.

| Activity | Board of Directors | Steering Committee | CISO / ISM | IT Operations | Legal / HR |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **Define Risk Appetite** | **A** | C | R | I | C |
| **Approve Security Policy** | **A** | R | C | I | C |
| **Develop Security Strategy** | I | C | **A / R** | C | I |
| **Implement Security Controls** | I | C | **A** | R | I |
| **Incident Response (Tactical)** | I | I | **A** | R | C |
| **Disaster Recovery Testing** | I | C | **A** | R | I |
| **Compliance & Regulatory Alignment** | A | C | R | I | **R** |

---

## 3. Governance Best Practices

* **Ultimate Accountability:** In CISM, the **Board of Directors** and **Senior Management** are ultimately accountable for the organization's security posture.
* **The "One A" Rule:** Ensure every row has exactly one **A**. Multiple accountabilities lead to a lack of ownership.
* **Alignment:** The RACI matrix must align the security program with business objectives to ensure governance is effective.
* **Separation of Duties (SoD):** Use the RACI to identify potential conflicts where the same person might be responsible for performing a task and accountable for auditing it.

---

## 4. CISM Exam Tips

1.  **Accountability vs. Responsibility:** The exam frequently tests the distinction. If a task fails, the person **Accountable** answers for it; the person **Responsible** did the work.
2.  **Risk Ownership:** The **Business Process Owner** is typically the Accountable party for risks associated with their specific department, while the CISO is Responsible for the risk management *process*.
